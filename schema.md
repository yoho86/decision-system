# 决策解剖 Schema（唯一权威定义）

每个决策 = 一个 markdown 文件，位于 `log/YYYY/YYYY-MM-DD-NNN-标题.md`。
`/decide` 和 `/review` 两个 skill 都以本文件为准。改 schema 就改这里。

**命名约定（方案 A）**：frontmatter 字段一律用**英文 key**（机器读，利于脚本校验与迁移）；正文小节一律用**中文标题**（人读的散文）。两类别混——展示时也别把"英文 key"和"中文小节"混列在同一栏。

**展示约定（title vs id）**：对用户**展示**决策时一律用 `title`（事件名），不要甩 `id`；`id` 是稳定的内部机器键，只用于存储、文件名、决策间交叉引用（如 `models.md` 的"用例"、正文"见 XXX"）。人读 title、机器认 id，两层分离——`id` 没有人类语义恰恰是它的优点。

## frontmatter 字段

| 字段 | 必填 | 取值 | 说明 |
|---|---|---|---|
| `id` | ✓ | `YYYY-MM-DD-NNN` | 当天序号，唯一 |
| `title` | ✓ | 文本 | 一句话标题 |
| `domain` | ✓ | 自由标签 | 如 投资 / 工作 / 健康 / 产品；仅用于切片，schema 本身通用 |
| `source` | △ | 文本/链接 | 决策来源溯源。从别的项目沉淀来的标这里，如 `prd:yourproduct/context/decisions.md#某决策行`；手记的可空 |
| `tier` | ✓ | `micro` \| `full` | 记录深度，由赌注决定 |
| `stakes` | ✓ | `low` \| `medium` \| `high` | 后果量级 |
| `reversibility` | ✓ | `reversible` \| `costly` \| `irreversible` | 贝佐斯 Type 2 / Type 1 |
| `state` | △ | 文本 | 决策当下的状态（疲惫/时间紧/情绪…，H.A.L.T.）。模式挖掘的金矿，强烈建议填 |
| `mode` | ✓ | `gut` \| `analysis` \| `rule` \| `external` \| `auto` | 你怎么决的（`auto` = 阶段4 AI 自主决策） |
| `criteria` | △ | 标签数组 | 这次取用了哪几条 value，如 `[现金流, 学习价值, 自主权]` |
| `models` | △ | 标签数组 | 这次推理用到的**思维模型**，如 `[机会成本, 沉没成本]`。用到即收录、理解在实践中收集（见 `models/models.md`）；`[]` = 没用任何模型，本身也是信号 |
| `chosen` | ✓ | 文本 | 最终选了什么 |
| `prediction` | ✓ | 文本 | 预测的结果（发动机之一） |
| `confidence` | ✓ | `0-100` | 对预测的信心百分比（发动机之二） |
| `horizon` | ✓ | `YYYY-MM-DD` | 结果何时揭晓，驱动复盘提醒（发动机之三） |
| `status` | ✓ | `open` \| `resolved` | 是否已回填结果 |

### 结果揭晓后回填（由 `/review` 填）

| 字段 | 取值 | 说明 |
|---|---|---|
| `outcome` | `as-expected` \| `better` \| `worse` \| `mixed` | 真实结果 vs 预测 |
| `resolved_date` | `YYYY-MM-DD` | 回填日期 |
| `process_score` | `1-5` | **抛开运气**，决策过程本身好不好 |
| `lesson` | 文本 | 一句话沉淀 |

**micro 决策**：只需 frontmatter，且只强制 `id/title/domain/tier/stakes/reversibility/chosen/prediction/confidence/horizon/status`。
**full 决策**：完整 frontmatter + 下方正文。

## 信心（confidence）怎么填

`confidence` = 你认为**你写的那条 `prediction` 会成真**的概率（0-100）。
不是"我多喜欢这个选择"、不是"这决定有多好"，而是"我赌的那个结果，实际发生的可能性多大"。
它是**校准引擎**的唯一钩子：攒够数据后系统回查——你说 70% 的事，是不是约 70% 真发生了？常年说高、实则偏低，就是系统性过度自信。

### 5 档锚点（等距对称，存代表值）

| 档 | 标签 | 存值 | 含义 / 下注测试 |
|---|---|---|---|
| ① | 搏赔率（长 shot） | `10` | 明知多半不中，为高赔率而搏；≈ 1:9 |
| ② | 不太可能（有戏） | `30` | 倾向不会，但有戏；≈ 3:7 |
| ③ | 拿不准（抛硬币） | `50` | 真不知道，二选一等于没信息；1:1 |
| ④ | 挺有把握（多半会） | `70` | 多半会，每 3 次错 1 次；≈ 7:3 |
| ⑤ | 很有把握（板上钉钉级） | `90` | 每 10 次错 1 次；9:1 |

- **锚点是辅助不是牢笼**：懒得想精确值时按档选；心里有更准的数（如 5% 的种子轮长 shot、95% 的 near-certain），直接手填档外值。校准时按最近的档归桶。
- **封顶 90**：永远别填 100（填了就是给自己挖坑）；真·near-certain 的罕见情况手填 95+。

### 校正与边界
- **下注测试**：信心 70 ≈ 你愿意以 7:3 赔率赌它成真；不愿意，说明没到 70。
- **翻转自检（可选，仅限二元对称决策）**：若这是个真·二元、赌注对称的判断（如"噪音可接受吗"），而你信心 <50，说明你其实更信反面——把预测改写成反面，信心跳到对称档（30↔70、10↔90）。
- ⚠️ **不对称下注不要翻转**：投资等"小概率搏大赔率"的决策，低信心（如 20）是**诚实且正确**的，翻转会抹掉决策本意。这种就大方填 sub-50。
- ⚠️ **赔率不进 confidence**：别因为"赔率好/值得赌"就把信心往上填。`confidence` 永远只是 P(预测成真) 这一件事；"值不值得下注"是 `process_score` + 正文推理的事。两者**正交**，校准才不被污染——一笔好赌注若落空，记 outcome `worse` + `process_score` 高分（好决策、正常的亏）。
- ⚠️ 前提：**预测要具体、能判定真假**，否则"成没成真"无法回答，信心就失去意义。
- ⚠️ **回溯/事后补记**的决策（结果已知才补填的），confidence 受后见之明污染，应在 `state` 或正文标注 `retrospective`，校准时排除。

## 正文结构（仅 full 需要）

```
## 情境/触发
什么事促成这个决策、当时手里有哪些信息。

## 备选项
列出考虑过的选项（含"什么都不做"），标出选了哪个。

## 推理与假设
- 关键假设：哪个假设一旦错了，这个决策就崩？
- 核心推理：为什么倾向这个选项。

## 预测
预测结果 + 信心 + 验证点（与 frontmatter 呼应，可展开细节）。

## 预演失败（premortem）
假设 N 个月后证明这是个错误决策，最可能的原因是？

## 复盘（结果揭晓后由 /review 回填）
- 预测 vs 现实
- 过程 vs 结果（结果坏 ≠ 决策错）
- 沉淀的规则候选
```

## 命名与 id 规则

- 文件名：`YYYY-MM-DD-NNN-简短标题.md`，标题用中文或拼音皆可，去空格。
- `NNN` 为当天序号，从 `001` 起。
- 跨年新建 `log/YYYY/` 目录。
