# 思维模型库（latticework）

> model = 你思考时借用的**透镜**（机会成本、复利、贝叶斯、二阶效应…）。
> 它不同于 `value`（你要什么）和 `rule`（你自己长出的经验）。三者区分见 `docs/specs/`。
>
> ⚠️ 这是**模板**。复制成你数据仓里的 `models/models.md` 后，「在库模型」从空开始——用到一个收一个，别批量灌入。

## 收录与理解（核心原则）

- **用到即收录**：模型在真实决策的推理里被用到（写进该决策的 `models` 字段）、且你确认确实用了，即进入模型库。始终自下而上、用到才收，**不批量灌入**。
- **理解在实践中收集，不预先声明**：不要求你上来写定义。每个模型的「应用中的理解」是**带日期的真实用法记录**——每次用它做决策，就记一句"这次你怎么用的"，理解从这些真实用法里**逐渐结晶**。
- **校准参照当外部锚点**：学科 / 权威定义并列存放，定期与你的实践用法比对——用法跑偏 = 理解有缺口，由 `/retro` 拎出来优化。确保**不只凭自我理解**。
- AI 只提议与记录，**绝不替你判定你懂了**；你的实践用法才是证据。

## 条目结构

```
名称 / 学科来源 /
应用中的理解(在实践中收集：带日期的真实用法 + 当前小结，由 /retro 随用法结晶) /
校准参照(权威/学科定义 + 关键澄清·外部锚点) /
触发信号(何时该想起它) / 典型误用与边界 / 关联模型(连成格栅) / 用例(决策 log 的 id)
```

下面给一个**示例条目**，展示结构长什么样（你自己的库从空开始，用到才长）：

### 机会成本（示例）
- 学科：经济学
- 应用中的理解（实践收集）：
  - YYYY-MM-DD · `<决策id>`：（这次你怎么用它的，一句话，带日期 + 链回决策）
  - 当前小结：⟨随用法增多，由 /retro 结晶；暂不预填⟩
- 校准参照：机会成本 = 你为此放弃的**次优选择**的价值；任何投入稀缺资源都有，**不限于"多个同级别选择"**——单一选项 / 什么都不做也算。
- 触发信号：任何投入时间 / 钱 / 注意力的取舍（含"看似没得选"时）
- 典型误用：只算投入成本，不算放弃的最好替代；误以为只有多选项才适用
- 关联模型：沉没成本
- 用例：（链回真实用到它的决策 id）
- 入库于：YYYY-MM-DD

## 养护（在 `/retro` 里做）

- **理解结晶**：把每个模型「应用中的理解」里的多条实践记录，归纳成一句"当前小结"（你确认才定）。理解来自实践积累，不是预先声明。
- **校准比对**：当前小结 vs 校准参照 vs 实战 `outcome`——常误用 = 理解有缺口，提议优化（你同意才改）；校准参照可由 AI / 学科补充，避免只凭自我理解。
- **效用分析**：哪些模型 → 高/低 `process_score`；多模型叠加（格栅效应）是否更优；凭直觉（`models: []`）的决策整体表现如何。
- 常用模型补踩坑；从不用的提议剪掉。模型增删一律**用户确认后**才动。
