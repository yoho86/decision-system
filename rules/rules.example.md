> rule = "在什么情况下我该怎么做"，是从决策数据里**长出来**、经你**验证**的 if-then 启发式。
> 与 value 的区别：rule 可被数据证伪，会随数据进化甚至退役；value 是你定的取向，不被数据改。
> 规则由 `/hyh-retro` 的周期复盘提出"候选"，你确认后才升为"已验证"。

## 规则是怎么诞生的（机制备忘）

1. `/hyh-retro` 周期复盘按 `state / mode / domain / stakes` 切片，找 `process_score` 系统性偏低/偏高的条件组合。
2. 写成 if-then **候选**，登记样本数与统计依据。
3. 你判断它是否合理（是否服务于 `values.md`）→ 升为"已验证"。
4. 每次复盘对已验证规则做回测；命中率持续下降 → 移到"已退役"。
